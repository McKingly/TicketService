Create Ticket

The service receives information and uses it to create a ticket. To do this a valid payment receipt has to be sent to the service.

The ticket is returned to the client and a new block is added to the blockchain.

GET /ticket/createTicket

Request

Content-type: application/json

{
    "secret": "secret",
    "details": {
        "purpose": "Transportation ticket",
        "company": "Transdev",
        "seat": "A45",
        "destinations": "Aveiro",
        "price": 0
    }
}
Field Description Type
secret Value to use when creating the hash String
details Aditional information about the ticket Object

Response

If successful, the response body contains a newly created instance of Ticket.

Content-type: application/json

{
    "ticket_id": 0,
    "timestamp": "2019-10-14T10:04:30.651Z",
    "details": {
        "purpose": "Transportation ticket",
        "company": "Transdev",
        "seat": "A45",
        "destinations": "Aveiro",
        "price": 0
    },
    "hash": "afd49088c68c2c17c329aedd6fbbe53a4d16b0ee8f187",
    "status": "inactive"
}
Field Meaning Type
ticket_id Ticket id number Integer
timestamp Timestamp for when the ticket was created Date
details Aditional information about the ticket Object
hash Hash to check ticket integraty String
status Specifies if the ticket is disabled, validated or expired String

Validate Ticket

The service extracts the ticket information from a QR code, checks to see if the integrity of the ticket is valid through the hash and returns a response based on the ticket's status.

If it hasn't been validated yet a new block will be added to the blockchain using the updated ticket information. If the ticket has already been validated no new block will be added.

PUT /ticket/validate/

Request

Content-type: application/json

{   
  "secret": "secret",
  "ticket": {
    "ticket_id": 0,
    "timestamp": "2019-10-14T10:03:57.756Z",
    "details": {
      "purpose": "Transportation ticket",
      "company": "Transdev",
      "seat": "A45",
      "destinations": "Aveiro",
      "price": 0
    },
    "hash": "afd49088c68c2c17c329aedd6fbbe53a4d16b0ee8f187",
    "status": "inactive"
  }
}
Field Meaning Type
secret Value to use when creating the hash String
ticket Ticket to be validated Object

Response

When validating a ticket, there are 2 different successful responses. When the ticket's status is inactive the response body contains an updated instance of the ticket with a new status value and hash.

Content-type: application/json

{
    "ticket_id": 0,
    "timestamp": "2019-10-14T10:04:30.666Z",
    "details": {
      "purpose": "Transportation ticket",
      "company": "Transdev",
      "seat": "A45",
      "destinations": "Aveiro",
      "price": 0
    },
    "hash": "afd49088c68c2c17c329aedd6fbbe53a4d16b0ee8f187",
    "status": "validated"
}
Field Meaning Type
ticket_id Ticket id number Integer
user_id User id number Integer
timestamp Timestamp for when the ticket was created Date
details Aditional information about the ticket Object
hash Hash to check ticket integraty String
status Specifies if the ticket is inactive, validated or expired String

When the ticket's status is validated an ok is returned.

Error Meaning
200 Ticket has been validated successfully.
201 Ticket is currently valid.

If the ticket can't be validated an error is returned.

Error Meaning
404 Invalid ticket
405 Ticket has expired
